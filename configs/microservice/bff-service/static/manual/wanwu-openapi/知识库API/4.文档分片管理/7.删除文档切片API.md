# 删除文档切片API

知识库删除文档切片支持对处理完成的文档删除指定分片。

## 请求接口

| Type     | Instructions                                                 |
| -------- | ------------------------------------------------------------ |
| 方法     | Http                                                         |
| 请求URL  | 按照应用实际API根地址，例如：<br />`http://localhost:8081/service/api/openapi/v1/knowledge/doc/segment/delete` |
| 调用方式 | 同步调用                                                      |
| 字符编码 | UTF-8                                                        |
| 请求类型 | DELETE                                                       |
| 鉴权方式 | `Authorization: Bearer {API Key}` |
| 请求格式 | `Content-Type: application/json`                             |
| 响应格式 | `Content-Type: application/json`                             |

## 请求参数

| Parameter    | Required  | Type   | Instructions                                                 |
|---------------|----| ------ |--------------------------------------------------------------|
| docId | 是  | string | 文档Id，仅支持处理完成状态的文档 |
| contentId | 是  | string | 分段内容Id，通过获取文档分片列表API获取 |

## 响应参数

| Parameter          | Type         | Instructions   |
|--------------------| ------------ |--------------|
| code               | int          | 状态码        |
| data               | object       | 返回数据   |
| msg                | string       | 返回信息       |

## 请求示例

```shell
curl --location --request DELETE 'http://localhost:8081/service/api/openapi/v1/knowledge/doc/segment/delete' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {API Key}' \
--data '{
    "docId": "2061726711199830016",
    "contentId": "68d24b24be7778c1e047b4c0e66c355c"
}'
```

## 响应示例

```json
{
  "code": 0,
  "data": null,
  "msg": ""
}
```

## 常见错误码

| Code   | Instructions                                                 |
|--------|--------------------------------------------------------------|
| 145013 | 删除文档分段失败，请稍后重试（如文档非处理完成状态） |
